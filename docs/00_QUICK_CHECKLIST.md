# Quick release checklist

## Repository and Pages

- [ ] The complete project—including the hidden `.github` folder—is committed to `main`.
- [ ] Repository **Settings → Pages → Source** is set to **GitHub Actions**.
- [ ] The most recent Actions run has green `build` and `deploy` jobs.
- [ ] The published URL opens without a 404.
- [ ] CSV download, automation report, filters, record dialogs, and source links work.

## Evidence quality

- [ ] Every material row has a source title, source URL, and verification date.
- [ ] Automated matches were manually checked for target specificity.
- [ ] Stage 6 and 7 rows use current official evidence—not an inferred trial status.
- [ ] Candidate aliases, sponsors, trial IDs, countries, populations, and serovars were checked.
- [ ] Discontinued, paused, completed, and active programmes are labelled accurately.
- [ ] Other registries and literature sources were searched where relevant.

## Governance

- [ ] Scope, inclusion criteria, and stage definitions are documented.
- [ ] A named data steward and review cadence are recorded.
- [ ] The dashboard displays an appropriate limitation statement.
- [ ] High-impact changes use a pull request or independent review.
- [ ] A dated snapshot or release is archived for analyses and publications.
